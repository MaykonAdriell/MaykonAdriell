## O que eu construo

- Produtos orientados a dados: dashboards, automações e fluxos decisórios
- IA como pipeline (não feature): classificação, roteamento, verificação e copilots
- Arquitetura por camadas (UI → hooks → services → data) com contratos explícitos e baixo acoplamento

---

## Princípios

- Systems-first: invariantes, limites e estrutura
- Complexity-aware: acoplamentos, trade-offs, intervenção mínima
- Pragmatismo > hype: medir, iterar, consolidar
- Contratos explícitos: types, schemas, RLS quando faz sentido
- Composição > acoplamento: módulos pequenos, responsabilidades claras
- Refatorar até ficar óbvio
